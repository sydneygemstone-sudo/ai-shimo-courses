# 毛球重构课｜教师启动说明

这是可直接上课的静态课程仓库，包含两套入口：

- 根目录 `index.html`：把原先前四节压成一节 60 分钟课；三年级高阶版先建立实模，再回头解释输入、数据和调试。
- `full-course/index.html`：原十节课统一重构版，每课带网页内互动，不依赖实体卡片、老师电脑或 AI使魔平台。

## 在线入口

- 三年级一小时高阶版：`https://sydneygemstone-sudo.github.io/ai-shimo-courses/`
- 完整十课互动版：`https://sydneygemstone-sudo.github.io/ai-shimo-courses/full-course/`
- 源码与版本历史：`https://github.com/sydneygemstone-sudo/ai-shimo-courses`

AI使魔训练营只需要跳转到上述 URL，不应复制课程 HTML 到平台内部。以后维护课程时，只修改这个仓库。

## 开课

1. 在 Mac 或 iPad 上打开同目录的 `index.html`。
2. 先点“播放本地测试音”，确认设备没有静音；课程本身不依赖音频。
3. 分别点开 AutoDraw 与 Quick, Draw! 做联网预检，再返回课件。
4. 按学生年龄选择路线。进入后可直接触控“上一页 / 下一页”，也可左右滑动。
5. 点“教师话术”查看当前页提示；点计时器开始 60 分钟倒计时。

无需启动 AI使魔平台，也无需登录毛球终端。平台可用时只作为最后的加分演示。

## 两条 60 分钟路线

| 时间 | 低龄优化版（6–8 岁） | 高龄加速版（9 岁+） |
|---|---|---|
| 0–14 分钟 | 毛球问题 + 乖乖机/聪聪机动作游戏 | 领取任务，直接开始毛球实模训练 |
| 14–22 分钟 | 用相机和听写理解“眼睛与耳朵” | 完成第一轮训练和新样本测试 |
| 22–32 分钟 | AutoDraw：慢慢补线索 | 输入管线与控制变量 |
| 32–43 分钟 | Quick, Draw!：20 秒猜画 | AutoDraw / Quick, Draw! 对抗实验 |
| 43–54 分钟 | 屏幕内水果数据训练/测试 | 数据、标签和失败案例调试 |
| 54–60 分钟 | 小驯养师复述与出口题 | 毛球产品挑战与测试清单 |

高龄路线满足“用一节课替代原四节入门”的目标；低龄路线保留故事和动作节奏，不强行压缩认知负担。

## 活动映射

- AutoDraw：`https://www.autodraw.com/`
- Quick, Draw!：`https://quickdraw.withgoogle.com/`

两个按钮在课件启动时会自动核对域名，并显示“两个活动入口已分离”。不要把 Quick, Draw! 替换成 AutoDraw；旧平台公共副本曾存在这一错误。

## 断网兜底

- AutoDraw 断网：点课件内“三步猜图”，页面会逐步揭示线索。
- Quick, Draw! 断网：点课件内本地抽题器，页面随机出题并完成 20 秒倒计时。
- 训练实验断网：直接使用课件内毛球实模训练台完成“训练—测试—找错—补例子”。
- 毛球终端不可用：教师直接扮演毛球，根据孩子给出的例子做一次正确猜测和一次错误猜测。

因此断网时仍可完成全部学习目标；联网游戏会被网页内本地实验替代，不需要实体卡片。

## iPad 开课前 smoke

- 建议横屏打开，确认页面没有被地址栏或 Home 指示条裁切。
- 低龄、高龄两个入口各点一次，确认都显示 `1 / 7`。
- 点“下一页”“上一页”、左右滑动各一次。
- 打开与关闭“教师话术”，确认文字可滚动。
- 启动倒计时，确认数字下降；再点一次暂停。
- 播放本地测试音，确认扬声器正常。
- 分别打开 AutoDraw 和 Quick, Draw!，确认进入不同网站，并练习从新标签返回课件。
- 临时断网刷新一次课件，确认两条路线与本地角色扮演内容仍可用。

## 维护与发布

修改后先运行 `node scripts/validate.mjs`。GitHub 每次推送也会自动执行同一检查；`main` 分支由 GitHub Pages 发布。

公开仓库只能包含本目录中的净化课件。不要把原 `ai-kids-course` 整树复制进来，因为原工作区包含服务配置、日志、profile、memory 和缓存。

## 教学边界

- 不把 AI 说成会思考的小人；统一说“根据见过的例子做预测”。
- 不要求孩子上传脸、声音或私人信息。
- Teachable Machine 只在老师的桌面设备已经提前验证时投屏演示，不作为这节课的必需环节。
- 毛球平台、数据库和编程游戏的修复不应阻塞本课。
