# 毛球十课公开版 QA 与重构裁决

日期：2026-07-14

## 结论

原十课的知识递进可以保留：

1. 看见学习：规则、传感器、数据、训练；
2. 用对 AI：历史、幻觉、情感与隐私边界；
3. 指挥创造：提示词、Agent、毕业项目。

但原 `courseware/lesson-01.html` 至 `lesson-10.html` 不能直接公开。它们目前更像教师投影讲稿，而不是可独立使用的网页课程。公开版必须重新建立交互、导航、平台边界和发布安全。

## 原版阻塞问题

- 十课没有真正的内页小游戏、表单或训练工具；AutoDraw、Quick, Draw!、Code.org 多数只是文字说明。
- 每课最后的“按 → 进入下一课”不会跨文件，脚本只会停在当前课最后一页。
- 原模板的嵌套 HTML 注释非法，会把模板残文暴露到页面或无障碍树。
- 触控体验不完整：教师话术和全屏依赖键盘，常见 iPad 竖屏会错过旧断点。
- 第 4 课从“训练模型”漂移成提前毕业/唤醒仪式，并依赖当天局域网、老师 Mac 和仍在开发的认画能力。
- 第 6、8 课依赖本机毛球终端或成人账号，无法由 GitHub Pages 单独提供。
- 第 9、10 课把尚未上线的毛球 Agent 写成真功能。公开版必须明确使用静态模拟器，不能伪装成后台真实执行。
- 原课程树含 `secrets.env`、聊天日志、memory、profile 与缓存。禁止整树复制到公开仓库。

## 用户本轮规则

- 取消“每课屏幕最多 20 分钟”的硬限制；页面可以成为主要授课介质。
- 取消实体指令卡、真假牌、菜单卡、监工卡、纸质时间线等前置材料。
- 课程不是 PDF；课程本体是带动画和互动的静态 HTML。
- 核心目标必须在无 Mac、无 MSI、无 AI使魔平台、无账号、无 API 的情况下完成。
- AI使魔训练营以后只跳转到公开静态课程 URL，不在平台内部复制维护另一版。
- 外部网站可以作为拓展，但不能承担核心验收。

## 十课重构映射

| 课次 | 保留的核心 | 网页内替代工具 | 去除的依赖 |
|---|---|---|---|
| 1 | 规则型 vs 学习型、AI 去魅 | 规则机/学习机快判 | 指令卡、教师白板 |
| 2 | 传感器、输入质量、权限 | 光线/噪声控制变量实验台 | 布、迷宫、方向卡、真实权限 |
| 3 | 数据、标签、多样性 | 猫咪数据集补课 | 水果卡、实体分类牌 |
| 4 | 训练—测试—调试闭环 | 毛球实模训练台 | 老师 Mac、局域网、认画后端 |
| 5 | AI 历史接力 | AI 时间穿梭机 | 年代牌、帘子、纸质时间线 |
| 6 | 幻觉与核查 | 毛球侦探局 | 真假牌、现场赌模型出错 |
| 7 | 情感和隐私边界 | 虚构消息分类器 | 真实隐私分享、安全卡打印 |
| 8 | 提示词和 teach-back | 愿望说明书工作台 | 三段卡、毛球终端 |
| 9 | Agent 循环与人类闸门 | 明确标注的静态计划模拟器 | 角色牌、监工卡、虚构真 Agent |
| 10 | 最小项目、验收、发布门 | 浏览器内毕业项目工作室 | 项目菜单卡、成人账号、装订材料 |

## 两套发布资产

### 三年级一小时高阶版

入口：仓库根 `index.html`

顺序调整为“先做后讲”：领取建模任务 → 网页内训练毛球 → 新样本测试 → 检查输入 → AutoDraw / Quick, Draw! 对抗实验 → 数据调试 → 产品出口题。

它用于 2026-07-15 直接上课，也是 AI使魔训练营的高阶入口。

### 完整十课互动版

入口：`full-course/index.html`

每课拥有：学习目标、网页内实验、60 分钟节奏、教师锚点、无需实体材料的替代方案、出口挑战和可选外部体验。完成进度只保存在当前浏览器 `localStorage`，不会上传学生输入。

## 发布前门槛

- 只提交当前净化仓库，不提交原 AI-Shimo 课程树。
- 自动检查两套 HTML 内联脚本语法、重复 ID、十课/十实验完整性、根路径资源和敏感引用。
- AutoDraw 必须只指向 `www.autodraw.com`；Quick, Draw! 必须只指向 `quickdraw.withgoogle.com`。
- GitHub Pages 开启 HTTPS；公开 URL 在 Mac、iPad 和普通未登录浏览器各打开一次。
- 真实设备 QA 后再把链接接入 AI使魔训练营。
